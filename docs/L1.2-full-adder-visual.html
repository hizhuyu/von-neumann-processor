为什么半加器不够用,全加器又是怎么拼出来的

1先看半加器为什么搞不定

半加器的能力就一条:算两个 1 位数,输出"和" + "进位"。

用半加器算 5 + 3 —— 看看它什么时候掉链子

二进制: 101 + 011 = 1000。从右往左一位位算。

第 1 列(最右边):1 + 1
1 + 1

半加器上场 → 本位 = 0,产生进位 = 1

1 0
↑ 送给下一列     ↑ 本位结果
第 2 列(中间):0 + 1 ← 进位 1 来了!
0 + 1 + 1 ← 上一列送来的

⚠️ 半加器只有两只手(A 和 B),接不住第三个"进位"!

真实答案是 2(二进制 10),但半加器只能给你 0+1=1。错了。

问题根源:多位加法从右往左算时,每一列都要接收右边那列送过来的"进位"。半加器接口不够,接不住这个进位 → 必须升级成"全加器",多一个输入口。

2全加器 = 半加器 + 一个进位入口

全加器跟半加器的区别就一点:多了一个输入口 Cin(carry in),用来接"上一列送来的进位"。

3 输入,2 输出(多了一个 Cin)

4怎么用两个半加器拼出全加器

思路就一句话:既然半加器只能算 2 bit,那就把 3 bit 拆成"先算前 2 个,再算后 2 个"。

Step 1:半加器 1 算 A + B

A
→
半加器 1 输入
B
→
半加器 1 输入

半加器 1 输出两个东西:S1(本位的和)和 C1(这一层产生的进位)。

Step 2:半加器 2 算 S1 + Cin

S1 (半加器1 输出)
→
半加器 2 输入
Cin
→
半加器 2 输入

半加器 2 输出两个东西:S(最终的本位)和 C2(这一层产生的进位)。

Step 3:Cout = C1 OR C2

两个进位 C1 和 C2 只要有一个是 1,最终就要进位。所以用 OR 门合并:

5真算一次:A=1, B=1, Cin=1

挑真值表最后一行 —— 半加器干不了的典型场景。

A=1, B=1, Cin=1,期望 S=1, Cout=1

▸ 半加器 1 算 1+1

S1 = 0
C1 = 1

S = 1
C2 = 0

▸ OR 合并 C1 + C2 = 1 + 0

Cout = 1
最终结果: CoutS = 11 = 3 ✓ 对照真值表,S=1, Cout=1

总结:全加器到底干了啥

1 个全加器 = 2 个半加器 + 1 个 OR 门

3 输入 (A, B, Cin) · 2 输出 (S, Cout) · 共 5 个门(XOR×2 + AND×2 + OR×1)

套路就一句话:先加两个,再加第三个。

  • 半加器 1:算 A + B → S1 + C1
  • 半加器 2:算 S1 + Cin → S + C2
  • OR 门:C1 + C2 → Cout(只要任一进位,就输出 1)

为啥要这个套路?

因为半加器接口不够,只能算 2 bit —— 我们手头又没有现成的"全加器零件",那就只能用现成的半加器去拼。这是工程上常见的"模块化复用"思路。
