# 📚 L1.1 半加器(Half Adder) 学习笔记

> 日期:2026-06-30
> 目标:理解"真值表 → 布尔表达式 → 电路"的完整思维流程

---

## 一、为什么需要"半加器"?

CPU 归根到底要会做加法,但 Logisim 给我们的最底层元件只有 **AND / OR / NOT** 这种布尔门 —— 它们不会"算"。

要算 `A + B` 这种最最最基本的加法,我们需要一个**最小单元**:
- 输入:两个 bit(`A`、`B`)
- 输出:这一位的**和**(`S`)+ **进位**(`C`)

这个最小单元就是 **半加器(Half Adder)** —— 它只算两 bit 相加,**不考虑上一位的进位**(那是全加器的事)。

---

## 二、真值表

把 A、B 所有可能的组合(共 4 种)列出来,写出期望的输出:

| A | B | S | C | 说明             |
|---|---|---|---|------------------|
| 0 | 0 | 0 | 0 | 0 + 0 = 00       |
| 0 | 1 | 1 | 0 | 0 + 1 = 01       |
| 1 | 0 | 1 | 0 | 1 + 0 = 01       |
| 1 | 1 | 0 | 1 | 1 + 1 = 10(进位!) |

> **为什么要 2 个输出?**
> `1 + 1 = 2`,二进制是 `10`,需要 2 个 bit 才能装下。所以低位是 `S`,高位是 `C`。

---

## 三、从真值表推导表达式

盯着 S、C 两列看规律:

- **S 列**:`0, 1, 1, 0` —— A、B **不一样的时候才是 1** → **XOR(异或)**
- **C 列**:`0, 0, 0, 1` —— A、B **都 1 的时候才是 1** → **AND(与)**

所以:

```
S = A XOR B
C = A AND B
```

**就这俩门搞定。**

## 四、Logisim 实现

### 4.1 元件清单

| 类型 | 数量 | 备注 |
|------|------|------|
| Input Pin | 2 | 标 A、B,facing=East(默认) |
| Output Pin | 2 | 标 S、C,**勾上 Output?**,facing=West |
| AND Gate | 1 | 接 A、B → C |
| XOR Gate | 1 | 接 A、B → S |

### 4.2 关键踩坑:Facing 方向

`Facing` 控制 Pin 上的**圆点(连接点)位置**。规则:

> **Pin 的圆点必须朝着电路那一侧**,不然接不上线。

- **输入 Pin** 在画布**左边**,电路在右边 → 圆点朝右 → `Facing = East`
- **输出 Pin** 在画布**右边**,电路在左边 → 圆点朝左 → `Facing = West`

❌ 错例:输出 Pin facing=East → 圆点在右(朝外),左边电路的线接不上
✅ 正例:输出 Pin facing=West → 圆点在左(朝内),电路侧线能接进去

> **注意:Facing 只影响视觉位置,不影响功能正确性。** 但方向不对的话,你**画线画不上去**,因为 Logisim 只允许从圆点那一侧引出线。

### 4.3 验证 checklist

用 Poke 工具拨 4 种组合,核对真值表:

- [ ] A=0, B=0 → S=0, C=0 ✓
- [ ] A=0, B=1 → S=1, C=0 ✓
- [ ] A=1, B=0 → S=1, C=0 ✓
- [ ] A=1, B=1 → S=0, C=1 ✓

4 个全对,半加器就算通了 ✅

## 五、我学到的核心方法论

**"真值表 → 表达式 → 电路"** 这三步是数字电路设计的标准流程:

1. **真值表**:穷举所有输入组合,列出期望输出 —— 这就是"需求"
2. **布尔表达式**:从真值表列里"读"出规律(看每列何时为 1)
3. **电路**:把表达式翻译成门电路 —— 这就是"实现"

这个流程**可以解决一切组合逻辑问题**,不只是加法器。后面所有的 MUX、ALU、译码器,本质都是这三步。
